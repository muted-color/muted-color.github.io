이 저장소는 아주 작은 단위의 연구 메모를 빠르게 쌓기 위한 GitHub Pages 템플릿으로 시작합니다.

## 이 블로그에 올릴 것

- 짧은 리서치 요약
- 실험 로그
- 읽은 자료 메모
- 구현 중 발견한 작은 인사이트

## 권장 글쓰기 패턴

1. 질문을 먼저 적기
2. 관찰한 사실만 짧게 적기
3. 임시 결론을 한 줄로 정리하기
4. 다음에 볼 항목을 남기기

## 권장 글 구조

새 연구 글은 필요에 따라 아래 흐름을 기본값으로 삼습니다.

1. 비교 대상 모델, 데이터셋, 논문을 `model-mention-cards`로 먼저 보여주기
2. 긴 실험 글은 앞쪽에 `## 요약`을 두고 4-6개 bullet로 결론과 주의점을 먼저 고정하기
3. 평가 기준이나 실험 조건을 `content-tabs`로 짧게 접어 정리하기
4. 평가 설계 요약을 `comparison-table`로 먼저 압축하기
5. 각 결과 섹션을 핵심 quote, figure/table, `sample-compare` 순서로 구성하기
6. 재현 조건은 `Appendix`의 comparison table에 모으기
7. 필요하면 `## References`, `## Experiment Resources`를 둔 뒤, 공개 글 마지막은 항상 `## Citation`으로 끝내기

## 문체 가이드

Gemma4 비교 글과 ESM2/LoRA 글의 공통 톤은 "담백한 연구 노트"입니다. 결과를 세게 팔기보다, 무엇을 봤고 어디까지 말할 수 있는지를 차분하게 좁힙니다.

### 문장 흐름

1. 문제 배경을 짧게 연다.
2. 글이 던지는 질문을 한 줄로 둔다.
3. `짧게 말하면` 또는 `답은 이렇게 정리된다`로 빠른 결론을 먼저 준다.
4. 바로 이어서 `다만`, `그래도`, `이 조건에서는`으로 해석 범위를 제한한다.
5. 본문에서는 `관찰 -> 해석 -> 주의점` 순서로 단락을 만든다.

### 자주 쓰는 표현

- `이 글은 ...를 비교한다.`
- `이 실험은 ...라는 질문을 던진다.`
- `짧게 말하면, 답은 이렇게 정리된다.`
- `...로 읽는 편이 맞다.`
- `...로 보는 편이 안전하다.`
- `...정도로 두는 편이 적당하다.`
- `...로 넓히면 과장된다.`
- `가장 안정적으로 남는 결론은 단순하다.`
- `실무적으로는 이렇게 쓰는 편이 좋다.`

### 피하는 표현

- 근거 없이 `압도적`, `완벽`, `명백히 SOTA`처럼 강하게 말하지 않습니다.
- `A가 B보다 좋다`에서 멈추지 않고, 어떤 지표/조건/seed/protocol에서 그런지 붙입니다.
- 자동 지표, judge 지표, runtime, variance를 한 덩어리 결론으로 섞지 않습니다.
- draft나 진행 중 실험은 최종 claim처럼 쓰지 않습니다.
- 좋은 결과만 남기고 반례, 약한 신호, 흔들린 지점을 지우지 않습니다.
- figure/table을 넣고 해석 없이 다음 섹션으로 넘어가지 않습니다.

### 단락 예시

```text
이 결과는 단순히 "LoRA가 이겼다"로 끝내기보다, 세 포인트로 읽는 편이 좋다. 방향성은 LoRA 쪽이지만, dataset마다 이유는 다르다. GB1에서는 frozen baseline이 이미 높아서 추가 이득이 protocol에 민감하고, AAV에서는 frozen estimate가 낮아 task-specific adaptation 여지가 더 커 보인다.
```

## 발행 전 소셜 썸네일 체크

- `image:`는 LinkedIn, Open Graph, Twitter/X 공유 썸네일용으로 반드시 둡니다.
- `hero_image:`는 본문 상단 hero 이미지용이므로 `image:`를 대체하지 않습니다.
- `image:`에는 SVG 대신 `1200x627` PNG 또는 JPG를 사용합니다.
- SVG hero를 쓰는 글은 같은 내용의 `social-thumbnail.png`를 따로 만들어 연결합니다.

## 실험 경로 메타데이터

- 실험 기반 글은 front matter에 `lab_path:`를 둡니다.
- `lab_path:`는 `~/lab/` 뒤쪽부터의 상대 경로만 씁니다.
- 예를 들어 실험 디렉토리가 `~/lab/experiment-lab/projects/flip-esm2-lowlabel-lora`라면 아래처럼 씁니다.

```yaml
lab_path: "experiment-lab/projects/flip-esm2-lowlabel-lora"
```

이 섹션은 이후 새 글을 쓸 때 자주 쓸 수 있는 서식을 한 페이지에서 빠르게 확인하기 위한 샘플입니다.[^sample-note]

### Reusable Components

현재 공통 CSS로 재사용할 수 있는 주요 컴포넌트입니다. 새 글에서는 필요한 블록만 복사해서 쓰면 됩니다.

- `model-mention-*`: 글 초반에 모델, 데이터셋, 논문 카드처럼 외부 리소스를 짧게 보여줄 때 사용합니다.
- `media-figure`: SVG, PNG, Plotly 같은 시각 자료를 본문 폭에 맞춰 보여줄 때 사용합니다.
- `table-figure`, `table-shell`, `metrics-table`, `comparison-table`: 캡션이 있는 표와 가로 스크롤이 필요한 표에 사용합니다.
- `content-tabs*`: 평가 지표, 실험 조건, 해석 가이드를 탭으로 나눠 보여줄 때 사용합니다.
- `sample-compare*`: 같은 질문에 대한 두 응답이나 두 조건의 실제 출력을 비교할 때 사용합니다.
- `metric-formula*`: 메트릭 정의, 식, 읽는 법을 짧은 key-value 블록으로 보여줄 때 사용합니다.
- `metric-detail__eyebrow`: 탭이나 설명 블록 안에서 Rubric, Formula, How to read 같은 작은 소제목을 표시할 때 사용합니다.
- `details-content`: Appendix처럼 보조 정보를 접어둘 때 사용합니다.
